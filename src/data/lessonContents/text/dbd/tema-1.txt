# Topic 1 · Database Management Systems (Sistemas de Gestión de Bases de Datos)

## Purpose / Propósito
- Review the technology underpinning modern Database Management Systems (DBMS). / Revisar la tecnología que sustenta a los Sistemas de Gestión de Bases de Datos (SGBD) modernos.
- Connect components, architectures, and operational objectives to understand their role in multi-user organizations. / Conectar componentes, arquitecturas y objetivos operativos para comprender su papel en organizaciones multiusuario.

## Objectives / Objetivos
- Identify essential properties of database technology and schema levels. / Identificar las propiedades esenciales de la tecnología de bases de datos y los niveles de esquema.
- Describe key DBMS functions and how they handle concurrency and data transfer. / Describir funciones clave de un SGBD y cómo gestiona concurrencia y transferencia de datos.
- Compare centralized and client-server architectures applied in real settings such as the BDASI lab. / Comparar arquitecturas centralizadas y cliente-servidor aplicadas en entornos reales como el laboratorio BDASI.
- List the priority objectives guiding DBMS design and operation. / Enumerar los objetivos prioritarios que guían el diseño y la operación de un SGBD.

## Activities / Actividades
### 1. Database Technology Overview (Panorama de la tecnología de bases de datos)
- Definition: structured, persistent collections stored in secondary memory. / Definición: colecciones estructuradas y persistentes almacenadas en memoria secundaria.
- Essential properties / Propiedades esenciales:
  - **Information system support:** centralizes and preserves organizational information. / **Soporte al sistema de información:** centraliza y preserva la información organizacional.
  - **Multi-user service:** enables safe concurrency and personalized views. / **Servicio multiusuario:** habilita concurrencia segura y vistas personalizadas.
  - **Data abstraction:** separates logical specifications from physical implementation. / **Abstracción de datos:** separa especificaciones lógicas de la implementación física.
  - **Integrity and security:** controls data quality, concurrent access, recovery, and authorization. / **Integridad y seguridad:** controla calidad de datos, acceso concurrente, recuperación y autorización.
- Data independence through levels / Independencia de datos mediante niveles:
  - **Logical schema**, **physical schema**, and **external schemas**. / **Esquema lógico**, **esquema físico** y **esquemas externos**.

### 2. Database Management Systems (Sistemas de Gestión de Bases de Datos)
- A DBMS implements a data model to create, query, and administer databases. / Un SGBD implementa un modelo de datos para crear, consultar y administrar bases.
- Historical models: hierarchical, network, relational, and object-relational. / Modelos históricos: jerárquico, en red, relacional y objeto-relacional.
- Core functions / Funciones fundamentales:
  - **Definition (DDL). / Definición (DDL).**
  - **Manipulation (DML). / Manipulación (DML).**
  - **Management (administration, security, backup, monitoring). / Gestión (administración, seguridad, backup, monitorización).**
- Concurrency and parallelism: coordinate multi-user transactions and leverage distributed parallelism when applicable. / Concurrencia y paralelismo: coordinación de transacciones multiusuario y uso de paralelismo distribuido cuando aplica.

### 3. Data Transfer Cycle (Ciclo de transferencia de datos)
- Storage hierarchy / Jerarquía de almacenamiento:
  - **Main memory and caches** (fast, limited capacity). / **Memoria principal y cachés** (rápidas, capacidad limitada).
  - **Secondary memory** (magnetic, optical, tape) with larger capacity. / **Memoria secundaria** (magnética, óptica, cinta) con mayor capacidad.
- Disk organization / Organización en disco:
  - Relations stored in record files; the transfer unit is the **block/page**. / Relaciones almacenadas en archivos de registros; la unidad de transferencia es el **bloque/página**.
- Query flow / Flujo de consulta:
  1. The user runs a query on their external view or the logical schema. / El usuario ejecuta una consulta sobre su vista externa o el esquema lógico.
  2. The DBMS translates the request and locates disk blocks. / El SGBD traduce la petición y localiza bloques en disco.
  3. It requests the OS to read and load data into main-memory buffers. / Solicita al SO la lectura y carga en buffers de memoria principal.
  4. Results are returned without changing the database state. / Devuelve resultados sin alterar el estado de la base.
- Update flow / Flujo de actualización:
  1. Repeat translation and reading. / Repite la traducción y lectura.
  2. Apply INSERT/DELETE/UPDATE in buffers. / Aplica INSERT/DELETE/UPDATE en buffers.
  3. Write modified blocks according to the transfer policy (possibly deferred). / Escribe bloques modificados según la política de transferencia (posible escritura diferida).

  Caption: Translation across schemas and block reading before executing SELECT. / Traducción entre esquemas y lectura de bloques antes de ejecutar SELECT.
- ![Deferred-write update cycle / Ciclo de actualización con escritura diferida](figure:dbd-tema-1/actualizacion)
  Caption: In-memory update with deferred block persistence. / Actualización en memoria y persistencia diferida de bloques.
- Operational keys: the DBMS manipulates only the assigned buffers; all I/O happens in block units. / Claves operativas: el SGBD sólo manipula los buffers asignados; todas las E/S se realizan por bloques.

### 4. Baseline Architectures (Arquitecturas básicas)
- **Centralized / Centralizada:** thin terminals connect to a server running applications, DBMS, and interfaces. / **Centralizada:** terminales ligeros conectados a un servidor que ejecuta aplicaciones, SGBD e interfaces.
- **General client-server / Cliente-servidor general:** specialized servers (web, mail, files, database) plus clients handling part of the logic. / **Cliente-servidor general:** servidores especializados (web, correo, archivos, base de datos) y clientes con parte de la lógica.
- **Database-oriented client-server / Cliente-servidor orientado a BD:** the client runs UI/logic (Visual, C++, Java) while the server processes SQL and transactions. / **Cliente-servidor orientado a BD:** el cliente ejecuta UI/lógica (Visual, C++, Java), el servidor procesa SQL y transacciones.
- BDASI example / Ejemplo BDASI:
  - SQL Developer as the Oracle client. / SQL Developer como cliente sobre Oracle.
  - Cycling database hosted on the lab server. / Base de datos Ciclismo en el servidor del laboratorio.
  - Shared logical schema, centrally managed physical schema. / Esquema lógico compartido, esquema físico administrado centralmente.

### 5. DBMS Objectives (Objetivos de un SGBD)
1. **Process concurrent transactions correctly / Procesar correctamente transacciones concurrentes:**
   - Database security (Topic 7). / Seguridad en bases de datos (Tema 7).
   - Transaction processing and integrity (Topic 2). / Procesamiento de transacciones e integridad (Tema 2).
   - Recovery and fault tolerance (Topic 3). / Recuperación y tolerancia a fallos (Tema 3).
   - Concurrency control (Topic 4). / Control de concurrencia (Tema 4).
2. **Process concurrently with efficiency / Procesar concurrentemente con eficiencia:**
   - Physical implementation (Topic 5). / Implementación física (Tema 5).
   - Query optimization (Topic 6). / Optimización de consultas (Tema 6).
   - Physical design of relational databases (Topic 8). / Diseño físico de bases relacionales (Tema 8).
